# Test Case Document: Product Detail Page - Interactive Elements & Rewards

| Field | Description |
| :--- | :--- |
| **Test Case ID** | TC_PDP_003 |
| **Test Scenario** | Verify product metadata, quantity controls, and auxiliary action links |
| **Test Case Description** | To validate that technical details (Reward Points, Product Code) are accurate and that the quantity increment/decrement buttons function correctly as per the UI image. |
| **Pre-Conditions** | User is on the Product Detail Page for "HTC Touch HD". |
| **Test Steps** | 1. Verify that the **Product Code** (Product 1) and **Reward Points** (400) are displayed. <br> 2. Check if the **Availability** label is styled with the "In Stock" red/green badge. <br> 3. Click the **"+" (Plus)** button next to the quantity input. <br> 4. Click the **"-" (Minus)** button next to the quantity input. <br> 5. Click on the **"Compare this product"** link. <br> 6. Verify the presence of the **"Size chart"**, **"Popup"**, and **"Ask Question"** action links. |
| **Test Data** | Product: `HTC Touch HD` |
| **Expected Result** | 1. Product Code displays "Product 1" and Reward Points display "400". <br> 2. Quantity increases by 1 when "+" is clicked and decreases by 1 when "-" is clicked (should not go below 1). <br> 3. "Compare this product" triggers a success notification or adds the item to the comparison list. <br> 4. Action links (Size chart, Popup, Ask Question) are clickable and open their respective modules. <br> 5. The "Brand" link (HTC) is clearly visible and correctly linked. |
