# Test Case Document: Product Detail Page - Quantity Counter

| Field | Description |
| :--- | :--- |
| **Test Case ID** | TC_PDP_002 |
| **Test Scenario** | Verify quantity update via increment (+) and decrement (-) buttons |
| **Test Case Description** | To ensure that the quantity value updates correctly when the user interacts with the UI toggle buttons. |
| **Pre-Conditions** | User is on the Product Detail Page of "HTC Touch HD". |
| **Test Steps** | 1. Locate the Quantity input field (default value should be 1). <br> 2. Click the **"+"** button once. <br> 3. Click the **"+"** button again. <br> 4. Click the **"-"** button once. <br> 5. Click the **"-"** button multiple times until it reaches 1, then click again. |
| **Test Data** | Product: `HTC Touch HD` |
| **Expected Result** | 1. Clicking **"+"** increases the value by 1 (e.g., from 1 to 2). <br> 2. Clicking **"-"** decreases the value by 1. <br> 3. The quantity should not be allowed to drop below 1. <br> 4. The value in the input field updates instantly without page refresh. |

| **Priority** | High |
| **Category** | Functional / UI |
