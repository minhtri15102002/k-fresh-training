# Test Case Document: Product Detail Page - Add to Cart Functionality

| Field | Description |
| :--- | :--- |
| **Test Case ID** | TC_PDP_002 |
| **Test Scenario** | Verify "Add to Cart" functionality with quantity and mandatory options |
| **Test Case Description** | To ensure that users can select a specific quantity, choose required product options (if any), and successfully add the product to the shopping cart with the correct total. |
| **Pre-Conditions** | 1. User is on a specific Product Detail Page (e.g., "HTC Touch HD"). <br> 2. Shopping cart is currently empty. |
| **Test Steps** | 1. Navigate to a product page: https://ecommerce-playground.lambdatest.io/index.php?route=product/product&product_id=28 <br> 2. Locate the "Qty" (Quantity) input field. <br> 3. Enter a valid quantity (e.g., "2"). <br> 4. Select any required options (Size, Color, etc.) if applicable to the product. <br> 5. Click the "Add to Cart" button. <br> 6. Observe the notification toast/popup. <br> 7. Click on the Cart icon/header to verify the item inside. |
| **Test Data** | Product: `HTC Touch HD`, Quantity: `2` |
| **Expected Result** | 1. The "Qty" field accepts numeric input and defaults to 1. <br> 2. A success message appears: "Success: You have added [Product Name] to your shopping cart!". <br> 3. The Cart total in the header updates immediately (Price x 2). <br> 4. The Mini-cart dropdown shows the correct product name, quantity, and unit price. <br> 5. If a mandatory option is missed, an error message "Option required!" should appear. |
